System Architecture Summary

Receiver Design

The receiver is built as an event-driven loop centered around a 4096-slot jitter buffer. Incoming UDP packets are parsed into either data packets or parity FEC packets. Data packets are stored by their sequence number, while parity packets are used to perform XOR-based recovery for any missing frames within a sliding window of (seq, seq-2) pairs.

Design Choices

  • Endianness: Strictly enforced using htonl() for transmission and ntohl() upon reception, ensuring cross-platform data integrity.
  • FEC Implementation: A pair-wise XOR parity scheme is used, triggered at seq % 4 == 2 (evens) and seq % 4 == 3 (odds), allowing for single-packet recovery.
  • NACK-based ARQ: Selective retransmission is managed by a gap scanner that monitors for missing sequence numbers. A cooldown mechanism (60ms) and RTT estimation are used to prevent NACK storms and network congestion.
  • Timing Synchronization: By calibrating against T0 and tracking the first received packet, the system maintains accurate playout deadlines despite potential drift.
  • Resource Management: Constant memory usage is maintained through pre-allocated circular buffers, ensuring efficient O(1) access to sequence slots and parity data.

System-Level Details

The system operates in two concurrent logical paths within a single loop using select(): the network reception path, which handles incoming traffic and FEC/NACK logic, and the playout path, which flushes completed frames to the application port. The implementation prioritizes stability by enforcing strict staleness and underflow guards, such as checking base < 2 in parity recovery and verifying that recovered members are not behind the playout_index.
